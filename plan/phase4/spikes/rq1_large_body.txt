- JSON stdlib has no size cap; anyio PIPE stream used by MCP SDK is byte-stream (no frame limit).
- Envelope overhead vs input grows because Cyrillic + emoji are escaped in default dumps.
- For 1 MB body input, request envelope is ~9,333,442 bytes.
- Claude CLI subprocess stdio PIPE: macOS default PIPE_BUF is 512 bytes but this is only
  for atomic writes; streaming writes chunk as needed.
- VERDICT: PASS — byte-identical roundtrip up to 4 MB.

Recommendation:
  MEMORY_MAX_BODY_BYTES default = 1_048_576 (1 MiB) is safe from transport perspective.
  The binding constraint is model context window + Telegram response size, NOT SDK frame.
  Documentation: cap body at 1 MiB; warn if write body > 256 KiB (single-turn context cost).
